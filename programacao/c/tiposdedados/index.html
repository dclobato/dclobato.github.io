    Tipos de dados em C¶

    No C, os tipos de dados definem a semântica, características de armazenamento e operações permitidas para elementos de dados, sendo expressos na sintaxe através de declarações de variáveis; a linguagem oferece tipos aritméticos básicos (inteiros e reais) e sintaxe para construir arranjos (arrays) e tipos compostos.

    O valor máximo que um determinado tipo de dados por armazenar depende de fatores como sistema operacional e tipo de processador. Uma forma de verificar os limites na máquina que você está usando, é verificar os valores disponíveis na biblioteca limits.h (veja aqui).

    Tipos principais¶

    A linguagem C fornece quatro especificadores de tipos dados aritméticos: char, int, float e double (além do tipo lógico bool), e os modificados signed, unsigned, short e long. A combinação dos tipos com os modificadores produz os seguintes tipos de dados.

    A relação entre os modificadores obriga que long long não seja menor que long, que não é menor que int, que não é menor que short. O menor tamanho é do tipo char.

    O tipo int deve ser o tipo inteiro que o processador alvo do programa compilado é mais eficiente manipulando. Isso permite grande flexibilidade: por exemplo, todos os tipos podem ser de 64 bits.

    O padrão C99 também adicionou os tipos complexos float _Complex, double _Complex, long double _Complex. Os tipos imaginários float _Imaginary, double _Imaginary e long double _Imaginary foram adicionados no padrão C11, junto com a biblioteca <complex.h> que permite acesso aos tipos complexo e imaginário.

    O tipo bool foi introduzido no padrão C99 (<stdbool.h>), e melhorado no C23, mas é importante notar que antes disso, inteiros eram usados para representar valores booleanos (0 para falso e qualquer outro valor para verdadeiro).

    Tipos de tamanho fixo

    Os tipos inteiros em C foram projetados para tornar o código portátil entre máquinas com diferentes tamanhos de palavra, o que significa que cada tipo pode ter diferentes intervalos de valores dependendo da arquitetura. No entanto, essa variação pode ser problemática quando um programa precisa operar com um intervalo específico de inteiros ou um tamanho fixo de armazenamento, independentemente da máquina em que será executado. A partir da versão C99 foram introduzidas definições de tipos que permitem declarar inteiros com tamanhos exatos, garantindo que o código-fonte do programa permaneça independente da arquitetura subjacente. Os tipos são os seguintes:

    • int\(b\)_t: inteiro com exatamente \(b\) bits e com sinal
    • uint\(b\)_t: inteiro com exatamente \(b\) bits e sem sinal

    Se não for necessário um tamanho específico, mas o menor tipo capaz de guardar um número de \(b\) bits, pode-se usar os seguintes tipos:

    • int_least\(b\)_t: inteiro com pelo menos \(b\) bits e com sinal
    • uint_least\(b\)_t: inteiro com pelo menos \(b\) bits e sem sinal

    Por questões de projeto do hardware do computador, alguns tamanhos podem ter um desempenho maior ou menor. Para isso, foram criados os tipos abaixo para representar os tipos de acesso mais rápido com pelo menos \(b\) bits.

    • int_fast\(b\)_t: inteiro de acesso mais rápido com pelo menos \(b\) bits e com sinal
    • uint_fast\(b\)_t: inteiro de acesso mais rápido com pelo menos \(b\) bits e sem sinal

    Para todos os casos, \(b \in \{8, 16, 32, 64\}\). Se por acaso a arquitetura não tiver um determinado tamanho, o tipo não estará definido no compilador, e um erro vai aparecer durante a compilação.

    Constantes¶

    Uma constante é um nome dado à variável cujos valores não podem ser alterados ou modificados. Uma constante é muito semelhante a variáveis em C, mas com a principal diferença que só pode ter o valor definido uma única vez, durante a declaração da variável. Isso significa que, uma vez que atribuímos um valor à constante, não podemos mudá-lo durante toda a execução do programa — ele permanece fixo.

    Veja bem, preste atenção!

    Na prática, o uso de const é preferível na grande maioria dos casos devido à segurança de tipo, possibilidade de depuração e escopo mais restrito. Use #define apenas se voce tiver uma razão muito forte para tal.

    Constantes podem ser definidas de duas formas: com o uso da chamada de pré-processador #define, ou com o uso da palavra chave const antes do tipo da variável durante a sua declaração. Do ponto de vista prático, as duas formas são equivalentes, apesar de "por baixo dos panos" serem completamente diferentes.

    Usando #define¶

    A diretiva #define é processada pelo pré-processador antes da compilação. O pré-processador substitui todas as ocorrências do nome da constante pelo seu valor antes do código ser compilado. Isso significa que:

    • A constante não tem um tipo associado. O valor substituído depende do contexto em que aparece no código;
    • Como a substituição ocorre antes da análise do código pelo compilador, erros podem ser mais difíceis de detectar, e;
    • Não ocupa espaço na memória, pois não é armazenada como uma variável real. Apenas ocorre uma substituição textual;

    Exemplo:

    #include <stdio.h>

#define PI 3.14159

int main() {
    double raio = 2.0;
    double area = PI * raio * raio; // O pré-processador substitui PI por 3.14159
    printf("Área: %f\n", area);
    return 0;
}

    No código acima, o pré-processador transforma PI * raio * raio em 3.14159 * raio * raio antes da compilação.

    Usando const¶

    O uso de const define uma variável que é tratada como uma constante pelo compilador. Diferente de #define, essa constante tem um tipo definido e ocupa espaço na memória (a menos que o compilador otimize e substitua seu uso diretamente).

    • A constante tem um tipo de dado explícito, permitindo verificações de tipo pelo compilador;
    • Pode ter um escopo definido, ao contrário de #define, que é global por padrão, e;
    • Pode ser usada para ponteiros, arranjos e outras estruturas de dados.

    Exemplo:

    const double PI = 3.14159; // Constante com tipo definido

    double area = PI * raio * raio;
    printf("Área: %f\n", area);
    return 0;
}

    Aqui, PI é tratado como uma variável que não pode ser modificada. Diferente de #define, ele tem um tipo e ocupa uma posição de memória.

    Enumerações¶

    As enumerações (ou enums) na linguagem C são um tipo de dados definido pelo usuário que consistem em um conjunto de constantes inteiras nomeadas. Elas fornecem uma maneira de criar nomes descritivos e significativos para um conjunto de valores inteiros, tornando o código mais legível e fácil de manter.

    Para declarar uma enumeração em C, utiliza-se a palavra-chave enum seguida pelo nome da enumeração e a lista de constantes entre chaves:

    1
2
3
4
5
    enum nome_enum {
    const1,
    const2,
    const3
};

    Por padrão, a primeira constante recebe o valor 0, a segunda recebe 1, e assim por diante. No entanto, é possível atribuir valores específicos às constantes, como no exemplo a seguir:

    1
2
3
4
5
6
    enum direcao {
    NORTE = 10,
    SUL = 20,
    LESTE = 30,
    OESTE = 40
};

    Após definir um tipo de enumeração, é possível declarar variáveis desse tipo:

    1
2
    enum direcao minhaVar;
minhaVar = LESTE; // Atribui o valor 30 à variável

    Características Importantes

    • Tamanho em Memória: Uma variável de enumeração ocupa o mesmo espaço que um inteiro (geralmente 4 bytes).
    • Uso em Expressões: As constantes de enumeração podem ser usadas em qualquer lugar onde a linguagem C permite uma expressão inteira.
    • Escopo: Os identificadores em uma lista de enumeração devem ser distintos de outros identificadores no mesmo escopo com a mesma visibilidade.

    Uma aplicação interessante das enumerações é o uso como flags (sinalizadores). Neste caso, cada constante é definida como uma potência de 2 para permitir a combinação de múltiplas flags usando operadores bit a bit.

    As enumerações são particularmente úteis quando:

    • Você tem um conjunto de valores relacionados que não mudarão, como dias da semana, meses, cores, etc.
    • Você precisa de constantes nomeadas para melhorar a legibilidade do código.
    • Você está trabalhando com flags e precisa combinar múltiplos valores.
    • Você quer evitar o uso de números mágicos no código.

    Embora seja possível realizar quase tudo em C sem usar enumerações, elas proporcionam uma maneira elegante e legível de trabalhar com conjuntos de constantes relacionadas, tornando o código mais fácil de entender e manter.

    Sequências de escape¶

    Alguns caracteres podem ser representados como sequências de escape: são conjuntos de dois ou mais caracteres que, na realidade, representam apenas um caractere. Um exemplo de sequência de escape é o \n, utilizado para representar o final de uma linha.

    Qualquer caracter pode ser representado pela sequência de escape \ooo, onde ooo é o número octal do caracter na tabela ASCII1. Outra possibilidade é representar o caracter utilizando a sequência de escape \xhh, onde hh é o número hexadecimal do caracter na tabela ASCII. Assim, a letra A pode ser armazenada como A, ou como \101, que é a representação em octal do valor 65 associado à letra A maiúscula na tabela ASCII, ou como \x41, que é a representação em hexadecimal de 65.

    Há algumas sequências de escape já definidas na linguagem C, para os caracteres mais comuns que não podem ser diretamente digitados.

    Sequência Significado
    \a Alerta (bipe, sino)
    \b Backspace
    \f Avanço/quebra de página
    \n Avanço de linha
    \r Retorno de carro
    \t Tabulação horizontal
    \v Tabulação vertical
    \\ Barra invertida (backslash)
    \' Apóstrofo, ou aspas simples
    \" Aspas duplas
    \? Ponto de interrogação
    \nnn O byte cujo valor numérico está representado como o número octal ooo
    \xhh O byte cujo valor numérico está representado como o número hexadecimal hh

    Arranjos (arrays)¶

    Para cada tipo T, exceto o tipo void, existe um tipo "arranjo de \(N\) elementos do tipo T". Um arranjo é uma coleção de valores, todos do mesmo tipo, armazenados de forma contígua na memória. Os elementos que compõem um arranjo de tamanho \(N\) são indexados por inteiros que variam de 0 até N-1. Por exemplo

    Declaração de arranjo unidimensional em C
    1
2
3
4
5
    int faltas[10];  // arranjo com 10 elementos, cada 
                 // um do tipo inteiro

c[3] = 8;  // O quarto elemento do arranjo recebe
           // o valor 8

    Arranjos com mais de uma dimensão (as vezes chamados de matrizes) são definidos como um "arranjo de arranjo ...", e todas as dimensões, exceto a mais externa, precisam ser valores constantes.

    Declaração de arranjo bidimensional em C
    1
2
    int a[10][8];  // arranjo de 10 elementos, cada um do tipo
               // 'arranjo de 8 elementos inteiros'

    Ponteiros¶

    Um ponteiro do tipo "ponteiro para T" é um tipo de dados que guarda um endereço de memória onde está armazenado um elemento do tipo T. Cada tipo de dado T tem um tipo de dados correspondente "ponteiro para T".

    Ponteiros são declarados com o uso de um asterísco (*) depois da declaração de tipo, e antecedendo o nome da variável.

    Declaração de ponteiros em C
    1
2
3
4
5
6
    char *letra;   // 'letra' guarda um endereço de memória onde está
               // armazenada uma variável do tipo char
long *salario; // 'salario' guarda um endereço de memória onde está
               // armazenada uma variável do tipo float
int *faltas;   // 'faltas' guarda um endereço de memória onde está
               // armazenada uma variável do tipo int

    Os ponteiros podem armazenar endereços de variáveis do tipo "arranjo de T" ou para estruturas, como no exemplo

    Ponteiros para arranjos e para estruturas em C
    1
2

    int *notas[10];  // Ponteiro para um arranjo de 10 valores inteiros

struct nodo {
    int valor;
    struct nodo *proximo;  // Ponteiro (endereço de memória que guarda
                           //  uma variável do tipo) para 'struct nodo'
};

    Ponteiros também podem ser declarados para variáveis do tipo "ponteiro para T", dando origem ao tipo "ponteiro para ponteiro para T", ou seja, uma variável que guarda um endereço de memória onde está armazenado um endereço de memória de uma variável do tipo T (uma indireção dupla). Os ponteiros para ponteiros são declarados com **.

    Declaração de ponteiro para ponteiro para inteiro
    1
    int **i;  // Um ponteiro para ponteiro para inteiro

    Em C, os operadores * e & desempenham papéis cruciais na manipulação de ponteiros, permitindo aos programadores interagir diretamente com a memória.

    O operador *, quando usado com um ponteiro, é conhecido como operador de indireção ou dereferência. Ele permite acessar o valor armazenado na posição de memória para a qual o ponteiro aponta. Em outras palavras, se um ponteiro p armazena o endereço de uma variável inteira x, então *p retorna o valor armazenado na variável x. Esse mecanismo é fundamental para modificar o valor de uma variável indiretamente, através de seu endereço de memória.

    Por outro lado, o operador & é o operador de endereço. Ele retorna o endereço de memória de uma variável. Ao aplicar & a uma variável, obtemos um ponteiro para essa variável. Por exemplo, se x é uma variável inteira, então &x retorna o endereço de memória onde x está armazenado. Esse operador é essencial para inicializar ponteiros, permitindo que eles apontem para variáveis específicas.

    A combinação de * e & possibilita a manipulação eficiente da memória, permitindo a criação de estruturas de dados dinâmicas e a implementação de algoritmos que exigem acesso direto à memória.

    Inicialmente, declaramos uma variável inteira chamada valor e a inicializamos com o valor 10. Em seguida, declaramos um ponteiro para inteiro chamado ptr e o inicializamos com o endereço de memória da variável valor, utilizando o operador &. Nesse ponto, ptr "aponta" para valor, ou seja, armazena o endereço de memória onde valor está armazenado. Em seguida declaramos um ponteiro para ponteiro para inteiro chamado ptr_ptr e o inicializamos com o endereço de memória do ponteiro ptr. Agora, ptr_ptr "aponta" para ptr, armazenando o endereço de memória onde ptr está armazenado.

    Perceba que ptr_ptr guarda um endereço de memória que armazena um endereço de memória para inteiro. Por isso é necessário obter o endereço da variável ptr com &ptr, ao invés de simplesmente fazer int **ptr_ptr = &ptr. A operação de atribuição int **ptr_ptr = &ptr está semânticamente incorreta, pois estamos atribuindo uma variável do tipo "ponteiro para inteiro" para uma variável do tipo "ponteiro para ponteiro para inteiro".

    A manipulação de ponteiros em C é frequentemente considerada um dos aspectos mais desafiadores da linguagem, gerando debates intensos entre programadores sobre sua complexidade. Essa característica fundamental da linguagem C tanto possibilita operações poderosas quanto introduz riscos significativos de erros. É importante entender que um "ponteiro para T" é uma variável como outra qualquer, que guarda um determinado valor (endereço de uma posição de memória) de um determinado tipo (posição de memória ocupada por um T). O fato de o "tipo" ser um "ponteiro para T" não faz a variável ser diferente de uma variável que guarda, por exemplo, um valor inteiro.

    A complexidade dos ponteiros em C manifesta-se principalmente através dos problemas que podem surgir quando mal utilizados. Um dos exemplos mais claros é a ocorrência de vazamentos de memória (memory leaks), que acontecem quando um programador, utilizando ponteiros, aloca memória dinamicamente, mas esquece de devolver essa memória ao sistema operacional antes de finalizar as atividade. Esse problema pode degradar o desempenho do programa ao longo do tempo, consumindo recursos do sistema de forma progressiva. Além disso, se o programar devolver a memória ao sistema operacional e tentar acessar a memória em seguida, temos um "ponteiro pendente" (dangling pointer): um ponteiro que aponta para uma área de memória que já foi liberada. As consequências disso não são previsíveis.

    Outro problema grave são as violações de acesso (access violations), que ocorrem quando um ponteiro tenta acessar um endereço de memória inexistente ou para o qual o programa não tem permissão. Como ponteiros são essencialmente números inteiros que podem ser manipulados através de aritmética, erros nos cálculos podem fazer com que apontem para locais desconhecidos na memória. Apontar para um local desconhecido ou sem autorização não é problema: o problema vai acontecer quando o ponteiros for dereferenciado.

    A aritmética de ponteiros, embora elegante e extremamente útil, é apontada como causa de uma falha de segurança importante na linguagem C. Teoricamente, a aritmética de ponteiros deve ser usada apenas dentro dos limites de um objeto alocado. Isso significa que, se você tem um arranjo de 10 elementos, você deve ajustar o ponteiro apenas para acessar esses 10 elementos. No entanto, C não realiza verificações automáticas de limites. Se um programador, por erro ou malícia, ajustar um ponteiro para além dos limites do objeto alocado, o sistema não impedirá essa ação, e quando um ponteiro ultrapassa os limites, ele pode sobrescrever dados de outros objetos que estão armazenados em áreas de memória adjacentes. Isso pode levar a corrupção de dados, falhas no programa ou vulnerabilidades de segurança, como a execução de código arbitrário.

    Por outro lado, a manipulação de ponteiros permite que os programadores escrevam código altamente eficiente, pois eles podem acessar e modificar dados diretamente na memória. Isso é particularmente útil em aplicações que exigem alto desempenho, como jogos e sistemas embarcados. Eles permitem a criação de estruturas de dados dinâmicas, como listas ligadas e árvores, que podem crescer ou diminuir de tamanho conforme necessário.

    Especialmente em sistemas embarcados, ou quando escrevendo programas que precisam manipular diretamente o hardware da máquina, os ponteiros são essenciais: eles permitem que os programadores leiam e escrevam em endereços de memória específicos que correspondem a registradores de hardware.

    Por fim, é só por causa da existência dos ponteiros que é possível passar parâmetros para funções por referência, permitindo que a função modifique os valores originais das variáveis.

    Uma abordagem disciplinada pode eliminar a maioria dos problemas: inicializar sempre os ponteiros, verificar se são nulos antes de dereferenciá-los e implementar uma estratégia clara de limpeza de memória

    Sinônimos de tipos (typedef)¶

    O comando typedef em C é usado para criar sinônimos ou "aliases" para tipos de dados existentes. Ele permite renomear tipos de dados, facilitando a organização e o entendimento do código.

    Veja bem, preste atenção!

    É importante ressaltar que o typedef não cria um novo tipo de dado, apenas permite que um tipo existente seja denominado de uma forma diferente

    A sintaxe básica do typedef é:

    1
    typedef <tipo_existente> <novo_nome>;

    O uso de typedef em programação oferece diversas vantagens. Em primeiro lugar, ele aprimora a legibilidade do código ao simplificar declarações complexas, facilitando o entendimento; em segundo lugar, facilita a manutenção, permitindo que a alteração de um tipo de dado seja feita em um único local, sem a necessidade de modificar cada instância de uso, e; por fim, ele simplifica as declarações de estruturas, reduzindo a necessidade de utilizar repetidamente a palavra-chave struct.
